# スタックちゃん ファームウェア 機能仕様

対象ファームウェア: `firmware/` (ESP-IDF 5.5.4 / ESP32-S3 / M5Stack CoreS3)
最終更新: 2026-06-27

---

## できること

### AI・会話

| 機能 | 詳細 |
|------|------|
| **AI 音声会話** | Xiaozhi クラウド AI（Qwen / DeepSeek LLM）とマイクで自然言語会話。WebSocket 経由でリアルタイムに音声をストリーミング受信 |
| **TTS 音声再生** | サーバー側で生成した Opus 音声をデバイスがスピーカーで再生 |
| **埋め込み音声の再生** | ファームウェアに焼き込んだ OGG/Opus ファイルを任意のタイミングで再生（`hal_bridge::app_play_sound()`） |

### HTTP API（ローカル & インターネット経由）

| エンドポイント | メソッド | 動作 |
|---------------|---------|------|
| `/hello` | GET | 「hello」音声を再生して `200 OK` を返す |
| `/github` | POST | GitHub Webhook を受信し「GitHubから通知です」と発話。JSON からリポジトリ名・送信者をログ出力 |

### ネットワーク

| 機能 | 詳細 |
|------|------|
| **WiFi 接続** | 2.4GHz WiFi、電波強度を 4 段階（None / Low / Medium / High）で取得 |
| **インターネット公開** | ngrok（固定ドメイン `unsold-promenade-askew.ngrok-free.dev`）で HTTP サーバーをインターネットに公開 |
| **GitHub Webhook 受信** | push / PR / issues など任意のイベントを受信して音声通知 |
| **BLE サーバー** | Bluetooth LE でアバター・モーション・RGB データを受信 |
| **ESP-NOW** | 近距離 ESP32 間の低遅延無線通信（別デバイスからの制御） |
| **OTA アップデート** | WiFi 経由でファームウェアを無線更新（手動トリガーのみ・自動実行なし） |

### ハードウェア制御

| 機能 | 詳細 |
|------|------|
| **アバター表示** | LCD に顔（目・口）を表示、表情変化（喜怒哀楽） |
| **サーボ制御** | 頭部の向き（上下・左右）を FTServo / SCServo ドライバで制御 |
| **タッチスクリーン** | 液晶タッチ入力の取得 |
| **RGB LED** | ボディ部の RGB LED をフルカラー制御 |
| **バッテリー監視** | 残量（%）と充電中かどうかの取得 |
| **バックライト輝度** | 液晶バックライトの明るさ調整（永続保存も可能） |
| **スピーカー音量** | 音量の取得・設定（永続保存も可能） |
| **IMU（BMI270）** | 加速度・ジャイロセンサー（傾き検出） |
| **カメラ（ハードウェア）** | StackChanCamera ドライバが組み込まれている |

### アプリ・システム

| 機能 | 詳細 |
|------|------|
| **アプリランチャー** | 画面タッチでアプリを切り替え |
| **ダンス機能** | BLE / WebSocket 経由でモーション指令を受けてサーボが動く |
| **セットアップ UI** | WiFi 設定・サーバー URL・音量などを画面から設定 |
| **EzData 連携** | M5Stack クラウドサービス（my.m5stack.com）との連携 |
| **時刻同期** | SNTP でインターネット時刻を取得、タイムゾーン設定可能 |
| **ウォームリブート** | アプリ状態を保ったまま再起動 |

## できないこと（現時点の制限）

### 音声・TTS

| 制限 | 理由 | 対応方針 |
|------|------|---------|
| **デバイス単体での任意テキスト発話** | ローカル TTS エンジンがない。発話はすべてサーバー経由か事前録音ファイルのみ | サーバーの TTS API が使えれば対応可能 |
| **HTTP API からテキストを渡して発話** | `/hello` や `/github` は固定音声のみ。動的なテキストには未対応 | エンドポイント拡張で対応可能 |
| **Webhook イベント別の音声差異** | push / PR / issues をすべて同じ「GitHubから通知です」で処理 | 音声ファイルを追加してイベント分岐で対応可能 |

### セキュリティ

| 制限 | 理由 | 対応方針 |
|------|------|---------|
| **Webhook 署名検証なし** | `/github` エンドポイントが `X-Hub-Signature-256` を検証していない | HMAC-SHA256 検証の実装で対応可能 |
| **HTTP API 認証なし** | `/hello` などに認証機構がない | API キーヘッダの検証を追加することで対応可能 |

### ネットワーク

| 制限 | 理由 | 対応方針 |
|------|------|---------|
| **ngrok の自動起動** | Mac 再起動後は手動で `ngrok` コマンドを実行する必要がある | launchd への登録で自動化可能（要ユーザー承認） |
| **オフライン AI 会話** | AI はクラウドサーバー（`47.113.125.164`）に依存 | ローカル LLM 対応は大規模な改修が必要 |
| **カメラ映像の HTTP 配信** | カメラドライバは組み込まれているが HTTP ストリーミングは未実装 | エンドポイント追加で対応可能 |

### ハードウェア

| 制限 | 理由 | 対応方針 |
|------|------|---------|
| **屋外・防水使用** | M5Stack CoreS3 は防水・防塵非対応（動作温度 0〜40°C） | 防水ケースの別途用意が必要 |
| **レーザー** | HAL に `setLaserEnabled()` はあるが、ハードウェア接続が必要 | 専用モジュール追加で対応可能 |
| **SD カード** | M5Stack CoreS3 に TF カードスロットが物理的に存在するが、現ファームウェアは未初期化・未使用 | SPI 初期化 + FAT マウントの実装で利用可能（会話ログ保存・動的音声ファイル読み込み等） |

## バックエンド Go サーバー (`server/`)

`http://47.113.125.164:12800` で稼働している M5Stack 公式バックエンド。Xiaozhi（AI 会話）とは別の独自サービス。**このサーバーは M5Stack が運営するため個人では変更・デプロイ不可**。自前で運用するには別途 VPS + MySQL 環境が必要。

### 役割

| カテゴリ | 内容 |
|---------|------|
| **デバイス管理** | MAC アドレスでデバイスを登録・ユーザーとバインド／アンバインド |
| **WebSocket 中継** | `/stackChan/ws` でデバイス↔スマホアプリ間の音声・モーション・通話状態をリアルタイム転送 |
| **AI 連携** | Xiaozhi の開発者トークンを取得・キャッシュし 24 時間ごとにリフレッシュ |
| **ダンス管理** | サーボ動作 JSON + BGM URL をデバイスごとに保存・配信 |
| **App Store** | 管理コンソールからファームウェア・アイコンをアップロードし配信 |
| **SNS 機能** | デバイス間のフレンド登録・投稿・コメント・パノラマ画像 |
| **OTA 配信** | ファームウェアの `.bin` ファイルをアップロードして配信（手動トリガーのみ） |
| **静的ファイル配信** | `/file/*` で音声・画像・ファームウェアを配信 |

### 認証方式

| ルート | 対象 | 認証 |
|--------|------|------|
| `/file/*` | 全員 | なし（公開） |
| `/stackChan/apps` | デバイス・全員 | なし（公開） |
| `/stackChan/*` | デバイス | RSA-OAEP 暗号化 MAC トークン |
| `/stackChan/v2/*` | スマホアプリ | JWT |
| `/admin/stackChan/*` | 管理コンソール | 管理者 JWT |

### GitHub Webhook を Go サーバー経由で実装する代替案

現在の実装（ngrok + デバイス側 HTTP サーバー）の代わりに、Go サーバーを介する構成も技術的に可能だった。

```
現在の構成:
  GitHub → ngrok → Mac → ESP32 HTTP サーバー

Go サーバー経由の代替構成:
  GitHub → Go サーバー /webhook/github（認証なし）
                  ↓ 既存の WebSocket 接続を通じて Push
               ESP32 が受信して音声再生
```

**メリット**: ngrok 不要・Mac が起動していなくても動作・URL が固定  
**デメリット**: 公式サーバー（`47.113.125.164`）は変更不可のため**自前 VPS + MySQL の構築が前提**。ESP32 側にも WebSocket メッセージ受信ハンドラの追加が必要

→ **自前サーバーを持たない場合、現在の ngrok 方式が最も手軽で現実的な選択。** デバイス単体で完結し、追加インフラ不要。

## SD カード

| 項目 | 状態 |
|------|------|
| **ハードウェア** | M5Stack CoreS3 に TF（microSD）カードスロット搭載（SPI 接続、GPIO 35/36/37/4） |
| **ファームウェア対応** | **未実装**。ピン定義なし、初期化コードなし |
| **現状** | 差し込んでいても認識されず、完全に未使用 |

### 活用できる用途（実装すれば）

| 用途 | 詳細 |
|------|------|
| 会話ログの保存 | Webhook 受信履歴・AI 会話内容をファイルに記録 |
| 動的音声ファイルの読み込み | OGG をファームウェアに埋め込まずカードから読み込み（容量制限の緩和） |
| GitHub イベントのペイロード保存 | 受信した JSON をそのまま記録 |
| カメラ撮影画像の保存 | カメラが有効化された際の写真保存先 |
